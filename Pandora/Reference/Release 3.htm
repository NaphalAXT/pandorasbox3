.:. Release 3 .:.

Changes:

  • Added option to modify all the link colors
  • Corrected command formats for Beta 36
  • Password protection: see BoxServer

Fixes:

  • Fixed hue picker crash when closing the window using the X button.
  • Fixed graphic glitches (hopefully) on deco tab

New Features:

  • Items tab
  • Notes tab
  • Modifiers support: some commands have a context menu that allows to use modifiers (self, multi, area and so on...). Global and Online trigger warnings, will add options to customize modifiers and warnings.

BoxServer:

  • Completely rewritten, now using a modular approach. In order to remove a feature from the BoxServer simply remove the folder that contains the scripts.
  • Password protection: now passwords are stored using a hash. The concept is very simple: when you enter your password Pandora will perform a hashing function and convert your password to another string (which you can view in the profile.xml file). The drill is that this string can not be reverted back because the hashing function does loose data about the original string. I'm told that no two strings produce the same result when elaborated through the hashing function. This hasn't been proved, but no two strings have produced the same results so far (and I mean in general, not in my tests), so this looks like the safest option to me. BoxServer will perform the same hashing on the password stored in the account file and compares the hashed strings to authenticate.
