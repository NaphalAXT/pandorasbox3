.:. Release 2 .:.

Changes:

  • Hue Picker:
    • No longer modal. You can now keep the hue picker open and use it while still using PB.
    • Added a Cancel button to exit without selecting a hue.
    • Added a Select button that will select the current hue on the PB main window
    • Now supports the zero hue.
    • Added a smell 0 link. Clicking it will set the current hue to zero aka no hue. 
  • Clicking the hue preview in the PB main window will not show/hide the hue picker rather than just opening it.
  • Hue will be added to the recently used hues only when using the Set button, not when selecting them through the hue picker or filling their value in.
  • Added a Clear button to the small Props tab (which of course clears all the fields).
  • Whenever two tree views are close to each other, they can now be resized.
  • Added an OK button to the options window.
  • Added dispaly of the art ID to the art viewer.
  • World map now displays spawns if SpawnData is enabled.

Fixes:

  • Pandora will no longer go offline when a requested datafile is not available on the server.
  • Getting SpawnData through BoxServer will now automatically enable spawns view on the travel map.
  • Tool tip for the tray icon now says 'Pandora's Box' rather than 'Tray'.
  • Show in taskbar option now correctly applied on startup.
  • Fixed bug where occasionally the window wouldn't be shown when clicking the tray icon.
  • Fixed bug where Pandora would minimize rather than closing when using Menu/Exit and having the Minimize with X button option checked.
  • When changing category on the travel tab, the map will no longer change its center to the first location listed in the previous category.
  • Fixed bug: Range value for spawns was wrongly updated using the amount value instead.
  • The button editor will now correctly localize the type of action associated with each button when opening on an existing button.
  • The world map will now correctly remember whether it was set to big or small.

New Features:

  • Props Tab
    • This tab shows all the properties that can be seen/set on the Props gumps for all the items and mobiles scripted on the server. This features relies on the PropsData.xml datafile, which can be generated using the [GenPropsData script.
    • Properties can be filtered according to the AccessLevel required to set them. (Admin will display all props, Councelor will display few props).
    • Show All Types: if checked this will display all types, even those that contain no additional properties (example: the Anvil item has no additional props, other than those declared for Item. If this option is unchecked only Item will be displayed, if it's checked both Item and Anvil will be).
    • Show All Props: if checked this will display all the properties available for a selected type (both inherited and declared properties). If unchecked, each entry will not display the properties of the base class, but only those declared by itself.
    • Find function: can search for a type. If the Show All Types option is unchecked, it will move to the parent class that declared at least one property.
    • Provides easy to use inputs for data types as TimeSpan, DateTime, Point3D and fast access to all the values in an enumeration.
  • Remote Scripts Explorer (BoxServer)
    • This feature allows a user to access one or more folders within the server's Scripts folder and upload, download, rename and delete files and folders.
    • Access to this feature must be specifies in the BoxRemoteExplorerConfig class (in other words, you must explicitly allow access for each account).
    • The explorer supports C#, VB, XML and text files.
  • Deco Tab
    • Same old deco tab, with a few more graphics (including the mahjong board, the only relevant addition).
    • Hued function works only with a custom script (BoxDecoItem.cs found in BoxServer\temp). This might be changed later, especially if I manage to get the RunUO devs to add a new constructor for the Item class.
    • A special 'Custom Deco' Category is now available: users with custom graphics can use it to add their own entries.
